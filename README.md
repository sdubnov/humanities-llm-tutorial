
# Humanities LLM Research Methods

A practical low-code / minimal-code course on using Large Language Models (LLMs)
for humanities, interpretive, and qualitative research.

This course teaches students how to move from casual AI chatting toward:
- structured prompting
- reproducible research workflows
- session memory and externalized cognition
- interpretive taxonomies and ontologies
- multi-turn interaction
- in-context learning
- role-based agents
- AI-assisted qualitative analysis

## Pedagogical Philosophy

The course treats:
- prompts
- rules
- annotations
- evaluations
- session summaries
- taxonomies

as scholarly research artifacts.

The emphasis is on:
- methodological transparency
- reproducibility
- critical interpretation
- controlled experimentation

rather than software engineering.

## Course Structure

### Lesson 0 — First Contact with LLMs
Students explore:
- ChatGPT
- Claude
- Gemini
- OpenAI Playground
- Hugging Face Spaces

They compare casual prompting with structured prompting and create their first session note.

### Lesson 1 — Environment Setup
Students:
- set up a local repository
- use markdown files
- learn session memory
- understand why research memory should be externalized

### Lesson 2 — Prompting as Method
Students:
- transform humanities questions into procedural prompts
- compare zero-shot and structured prompting
- iteratively revise prompts
- evaluate outputs critically

### Lesson 3 — Sessions, CLI, Memory, and Commands
Students compare:
- continuing a session
- restarting from markdown summaries
- fresh sessions

Topics:
- context windows
- hidden compression
- reproducibility
- restartability
- external memory

### Lesson 4 — Humanities Concepts, Taxonomy, Ontology
Students:
- extract analytical concepts from theory texts
- build taxonomies and ontologies
- operationalize literary concepts
- apply categories to poems and lyrics

### Lesson 5 — Multi-Turn Interaction and In-Context Learning
Students learn:
- multi-turn interaction
- in-context learning
- teacher-annotated examples
- chain-of-thought style procedures
- prompt iteration

### Lesson 6 — Agent Roles for Interpretive Analysis
Students explore role-based agents:
- Theorist
- Analyst
- Skeptic
- Editor
- Archivist

They compare single-agent and multi-agent workflows.

## Final Capstone

The capstone integrates:
theory → taxonomy → prompting → multi-turn analysis → evaluation → revision.
