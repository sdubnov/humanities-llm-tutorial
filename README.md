# Humanities LLM Research Methods

A practical low-code / minimal-code course on using Large Language Models (LLMs) for humanities, interpretive, and qualitative research.

Created by Shlomo Dubnov  
sdubnov@ucsd.edu

This course teaches students how to move from casual AI chatting toward:

- structured prompting
- reproducible research workflows
- session memory and externalized cognition
- interpretive taxonomies and ontologies
- multi-turn interaction
- in-context learning
- explicit reasoning traces
- role-based agents
- AI-assisted qualitative analysis

## Pedagogical Philosophy

The course treats:

- prompts
- rules
- annotations
- evaluations
- session summaries
- taxonomies
- reasoning traces

as scholarly research artifacts.

The emphasis is on:

- methodological transparency
- reproducibility
- critical interpretation
- controlled experimentation
- visible evidence-to-claim reasoning

rather than software engineering.

## Core Progression

The course follows this progression:

```text
chatting → prompting → files → rules → taxonomies → multi-turn analysis → explicit reasoning traces → role-based agents → research workflows
```

This progression moves students from casual interaction with LLMs toward reproducible humanities research methods.

“Explicit reasoning traces” refers to visible evidence-to-claim reasoning, branching interpretation, and procedural analysis. “Role-based agents” refers to structured research roles such as Theorist, Analyst, Skeptic, Comparativist, Archivist, and Editor, rather than fully autonomous software agents.

## Course Structure

### Lesson 0 — First Contact with LLMs

Students explore:

- ChatGPT
- Claude
- Gemini
- OpenAI Playground
- Hugging Face Spaces

They compare casual prompting with structured prompting and create their first session note.

### Lesson 1 — Environment Setup

Students:

- set up a local repository
- use markdown files
- learn session memory
- understand why research memory should be externalized

### Lesson 2 — Prompting as Method

Students:

- transform humanities questions into procedural prompts
- compare zero-shot and structured prompting
- iteratively revise prompts
- evaluate outputs critically

### Lesson 3 — Sessions, CLI, Memory, and Commands

Students compare:

- continuing a session
- restarting from markdown summaries
- fresh sessions

Topics:

- context windows
- hidden compression
- reproducibility
- restartability
- external memory

### Lesson 4 — Humanities Concepts, Taxonomy, Ontology

Students:

- extract analytical concepts from theory texts
- build taxonomies and ontologies
- operationalize literary concepts
- apply categories to poems and lyrics

### Lesson 5 — Multi-Turn Interaction and In-Context Learning

Students learn:

- multi-turn interaction
- in-context learning
- teacher-annotated examples
- critique loops
- prompt iteration
- taxonomy revision

### Lesson 6 — Reasoning Traces: CoT, ToT, and PoT

Students learn how to make interpretive reasoning explicit, inspectable, and revisable.

Topics:

- Chain-of-Thought-style interpretive traces
- Tree-of-Thought-style branching interpretation
- Program-of-Thought-style procedural analysis
- evidence-to-claim reasoning
- uncertainty tracking
- comparison of direct prompting and structured reasoning

### Lesson 7 — Agent Roles for Interpretive Analysis

Students explore role-based agents:

- Theorist
- Analyst
- Skeptic
- Comparativist
- Editor
- Archivist

They compare single-agent and multi-agent workflows. Agent roles are introduced after reasoning traces so that each role works with explicit evidence, reasoning steps, and uncertainty notes.

## Final Capstone

The capstone integrates:

```text
theory → taxonomy → prompting → multi-turn analysis → reasoning traces → role-based critique → evaluation → revision
```

Students submit a reproducible humanities LLM workflow, including saved prompts, outputs, reasoning traces, critiques, revisions, and a final methodological reflection.
