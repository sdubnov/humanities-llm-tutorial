# Instructor Solution and Run Instructions

This file gives an instructor-facing walkthrough for running the project exactly as provided, verifying the sample output, and explaining how the sample materials were derived.

## 1. What this project demonstrates

The project demonstrates a basic agentic humanities workflow:

```text
theory concept -> agent role -> structured prompt -> score matrix -> Python report -> methodological critique
```

The topic is not simply "postmodern books." The topic is the methodological problem of making interpretive categories visible enough to compare, while still treating them as debatable humanities constructs.

The sample workflow asks:

> How do different postmodern texts distribute their "vibe" across ontological instability, paranoia/affective density, and media/structural friction?

## 3. How the sample materials were derived

### Agent roles

The three agent roles come from the original prompt-dump idea of simulating a seminar room rather than using one generic model response. The roles were normalized into reusable configuration objects:

- Ontological Tracker: reality rules, worlds, time, boundaries, fiction/reality collapse.
- Affective Vibe-Scout: paranoia, exhaustion, irony, manic play, detachment.
- Media Formalist: typography, footnotes, layout, recursive framing, text-as-object.

### `sample_matrix.csv`

The sample matrix is an instructor-created demonstration, not a benchmark dataset. It uses short descriptions of five well-known postmodern novels and assigns plausible scores on three 1-10 dimensions:

- `ontological_multiplicity`
- `paranoia_affective_density`
- `media_structural_friction`

The values are intentionally debatable. For example, *Libra* receives high paranoia but low ontological multiplicity; *House of Leaves* receives high scores on ontology and media friction; *Slaughterhouse-Five* receives high ontology but moderate media friction.

The most important column is `reliability_warning`. It reminds students that summary-based scoring may reflect reputation and canon knowledge more than direct textual evidence.

### `run_analysis.py`

The script is intentionally simple. It does not perform machine learning. It reads the CSV, validates that scores are between 1 and 10, applies transparent threshold rules, and prints a report.

The cluster rules are heuristic:

```text
media >= 8 and ontology >= 7       -> Ergodic / Media Monsters
affect >= 8 and ontology <= 5      -> Paranoid Realists
ontology >= 8 and media <= 6       -> Chrono-Fractured Empaths
affect >= 8 and media >= 8         -> Overload Systems
otherwise                          -> Hybrid / Unsettled Postmodern Form
```

These labels were adapted from the prompt-dump examples, where texts were grouped into categories such as Paranoid Realists, Ergodic Monsters, and Chrono-Fractured Empaths. The labels should be treated as classroom discussion starters, not final categories.

### `sample_report.txt`

This is the expected report generated by running:

```bash
python run_analysis.py sample_matrix.csv --output sample_report.txt
```

If the script and matrix are unchanged, the newly generated report should match the provided sample report.

### `rubric.md`

The rubric is derived from the project learning goals. It evaluates whether students can design bounded agents, translate theory into metrics, construct a reliable matrix, run and interpret the script, and critique the limits of LLM-generated interpretation.

## 4. Running the instructor solution locally

### Option A: You already have the course repository locally

From the root of your local repository:

```bash
cd projects/postmodern-vibe-agentic-matrixing
python run_analysis.py
```

To save a report:

```bash
python run_analysis.py sample_matrix.csv --output instructor_test_report.txt
```

Compare the output with:

```bash
cat sample_report.txt
```

### Option B: Download only this project folder, not the whole repository

Use Git sparse checkout:

```bash
mkdir postmodern-vibe-test
cd postmodern-vibe-test

git init
git remote add origin https://github.com/sdubnov/humanities-llm-tutorial.git
git sparse-checkout init --cone
git sparse-checkout set projects/postmodern-vibe-agentic-matrixing
git pull origin main

cd projects/postmodern-vibe-agentic-matrixing
python run_analysis.py
```

This gives you only the selected project path rather than checking out the whole repository.

### Option C: Ask Codex to create the folder from copied materials

In Codex, give an instruction like this:

```text
Create a folder called projects/postmodern-vibe-agentic-matrixing/. Add the files I provide one by one. Preserve the filenames exactly. After creating the files, run: python projects/postmodern-vibe-agentic-matrixing/run_analysis.py. If the script runs successfully, save the output as projects/postmodern-vibe-agentic-matrixing/instructor_test_report.txt and compare it to sample_report.txt.
```

## 5. Suggested instructor demo script

1. Open `project_background.md` and explain the analysis challenge.
2. Open `agents_config.json` and show how the agents divide interpretive labor.
3. Open `sample_matrix.csv` and point out that the scores are debatable.
4. Run:

```bash
python run_analysis.py
```

5. Ask students which cluster seems persuasive and which seems suspicious.
6. Open `sample_report.txt` and show that the report is generated from the matrix, not from hidden model reasoning.
7. Ask students to revise one agent, rescore one row, rerun the script, and explain what changed.

## 6. Expected output summary

When run on the sample matrix, the script should assign:

| Text | Expected cluster |
|---|---|
| *Libra* | Paranoid Realists |
| *House of Leaves* | Ergodic / Media Monsters |
| *Gravity's Rainbow* | Ergodic / Media Monsters |
| *Infinite Jest* | Overload Systems |
| *Slaughterhouse-Five* | Chrono-Fractured Empaths |

Class-level averages should be approximately:

```text
Average ontological_multiplicity: 7.20
Average paranoia_affective_density: 8.60
Average media_structural_friction: 7.20
```

## 7. How to use this as a solution key

Treat `sample_matrix.csv` and `sample_report.txt` as a solution demonstration, not an answer key. Student results may differ if they rewrite agents, use different text descriptions, or add more evidence. The quality of the project depends less on matching the sample and more on whether the student can explain the consequences of their methodological choices.
