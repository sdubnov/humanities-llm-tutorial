# Project: Postmodern Vibe Agentic Matrixing

## Placement in the course

**Recommended repository path:**

```text
projects/postmodern-vibe-agentic-matrixing/
```

This is an **upper-intermediate introductory-course project**, not yet a full advanced-course project. It should be taught after the basic lessons on prompting, taxonomies, multi-turn interaction, and agent roles.

The project acts as a bridge between the introductory course and the advanced course. It introduces students to matrix-based, agentic humanities analysis without yet requiring RAG, evaluation harnesses, fine-tuning, or a full research platform.

## Project overview

This project addresses a methodological problem: literary scholars often use rich but slippery interpretive terms such as “vibe,” “paranoia,” “fragmentation,” “media friction,” or “ontological instability.” These terms are meaningful, but they are difficult to compare across texts unless the researcher makes the categories, evidence, and assumptions explicit.

Students build a small, reproducible workflow for analyzing the “vibe” of postmodern literary works. Instead of asking one model to produce a single summary or interpretation, students divide the task among role-based analytical agents:

1. **Ontological Tracker** — tracks instability in worlds, reality rules, temporal structure, fiction/reality boundaries, and multi-world narration.
2. **Affective Vibe-Scout** — tracks paranoia, exhaustion, irony, manic play, emotional detachment, and broader structural affect.
3. **Media Formalist** — tracks typography, footnotes, nonlinear layout, recursive framing, physical reading friction, and text-as-object effects.

The agents produce numerical scores and short qualitative justifications. Students collect the outputs into a matrix, run a simple Python script to assign heuristic clusters, and then critique the method.

## Learning goals

By the end of this project, students should be able to:

- design role-specific LLM agents for interpretive analysis;
- translate abstract humanities concepts into usable analytical dimensions;
- distinguish a prompt, an agent role, a corpus, a matrix, and an evaluation artifact;
- create a reproducible qualitative-analysis workflow using saved prompts and CSV files;
- use a simple Python script to summarize and cluster interpretive scores;
- critique LLM outputs for summary bias, reputation bias, flattening, and false precision;
- explain why a matrix is not a replacement for interpretation but a tool for provoking better interpretation.

## Prerequisites

Students should complete the following lessons first:

- **Lesson 1 — Environment Setup**: working with markdown files, folders, and saved research artifacts.
- **Lesson 2 — Prompting as Method**: turning interpretive questions into structured prompts.
- **Lesson 4 — Humanities Concepts, Taxonomy, Ontology**: converting theory into categories and analytical dimensions.
- **Lesson 5 — Multi-Turn Interaction and In-Context Learning**: revising prompts, comparing outputs, and stabilizing behavior through examples.
- **Lesson 6 — Agent Roles for Interpretive Analysis**: comparing single-agent and multi-agent workflows.

Recommended but optional:

- **Lesson 3 — Sessions, CLI, Memory, and Commands**: useful for running `run_analysis.py` locally and preserving outputs across sessions.

## Suggested class workflow

### Session 1 — From theory to agents

1. Read `project_background.md` to frame the central analysis challenge.
2. Introduce the problem: “vibe” is real as an interpretive intuition but unstable as a category.
2. Discuss why a single LLM answer is not methodologically transparent.
3. Review the three agent roles in `agents_config.json`.
4. Have students revise one agent instruction to make it more precise.
5. Run one shared text through all three agent lenses.
6. Compare differences in outputs.

### Session 2 — Matrixing and critique

1. Students run the agents on a small corpus of 5–8 texts.
2. They enter scores and justifications into `corpus_template.csv`.
3. They run `run_analysis.py` on the completed matrix.
4. They compare machine-generated clusters with their own literary intuitions.
5. They write a short critique identifying where the method helped and where it distorted the texts.

## Basic mode and extended mode

### Basic mode

Students manually run prompts in a chat interface and fill in the matrix by hand.

### Extended mode

Students run the Python script, modify the cluster thresholds, add a fourth agent, or compare different models.

## Methodological cautions

Students should explicitly discuss:

- **Summary bias**: the model may be analyzing a summary, a review, or its prior knowledge of a famous work rather than evidence from the provided text.
- **Reputation bias**: famous books may receive canonical scores because of how they are discussed online.
- **False precision**: a 1–10 number may look objective even when it is interpretive.
- **Flattening**: words like “absurd,” “ironic,” or “postmodern” may become clichés rather than meaningful analysis.
- **Agent drift**: agents may ignore their assigned lens and produce general summaries.

## Deliverables

Each student or group submits:

1. one revised agent prompt;
2. a completed matrix for at least five texts;
3. the report generated by `run_analysis.py` or a manually written equivalent;
4. a short methodological reflection explaining what the matrix revealed and what it concealed.
