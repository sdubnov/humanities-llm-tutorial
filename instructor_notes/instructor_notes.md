
# Instructor Notes and Design Decisions

## Core Educational Goal

The course teaches students to treat LLM interaction as a reproducible research methodology rather than casual chat.

## Externalized Cognition

Students learn to externalize:
- prompts
- rules
- evaluations
- summaries
- taxonomies

into markdown files.

## Why Manual Session Notes?

LLM memory is temporary, opaque, and platform-dependent.
Research memory should be:
- persistent
- inspectable
- portable
- revisable

## Why Compare Same Chat vs Restart?

Students directly experience:
- hidden context
- session compression
- instability
- reproducibility issues

## What Is an Agent Here?

This course uses pedagogical agents:
role + task + perspective + output format.

Not orchestration systems.

## Why Multi-Turn Interaction?

Multi-turn interaction gradually specializes the model through:
- examples
- corrections
- taxonomies
- evaluations
- shared terminology

without requiring fine-tuning.

## Prompting vs Fine-Tuning

Prompting:
- visible
- editable
- flexible
- low-cost

Fine-tuning:
- changes model behavior
- requires datasets
- harder to inspect
- more infrastructure

## Recommended Classroom Emphasis

Students should:
- critique outputs
- identify overinterpretation
- revise prompts
- compare workflows
- understand limits of LLMs

rather than merely obtaining plausible answers.
